# PRD · Council AI 健康助手（C 端）

> 版本：v0.1（初稿，待评审）
> 一句话定位：**一间"看得见专家讨论"的 AI 健康会诊室 + 会记住你的健康档案。**
> 技术内核：异构多 Agent 共识引擎 · 纵向个人健康记忆

---

## 1. 为什么做 / 差异化策略（先读这一段）

竞品调研结论：查报告、报告解读、用药、症状问诊、找医生**已是行业标配**，讯飞晓医 / 百度灵医 / 蚂蚁阿福功能高度同质。**功能清单上无法取胜。**

我们只押两个别人没做、且与我们技术内核咬合的差异点：

| 差异点 | 别人现状 | 我们的做法 |
|---|---|---|
| **A. 透明的多专家共识** | 全是单模型"一个答案" | 多个异构专科 Agent 各自分析、辩论、给论据，**透明呈现共识 + 分歧 + 不确定性** |
| **B. 纵向健康记忆** | 阿福有档案/日记，但偏"存储" | AI **主动联动历史**："你一个月前也有类似症状"，并跨时间推断可能病因 |
| **C（增强）. 中西医双视角** | 讯飞有"中医辨证"但孤立 | 中医作为共识面板里的**一个独立视角**，与西医同台，呈现两种思路的异同 |

> 原则：**不打流量战（打不过阿福的医+药+险闭环），打"可信度 + 个性化记忆"战。** 用户感知是"它更像一群医生认真讨论我的情况，而且它还记得我"。

## 2. 两个核心场景（产品脊梁：诊后 + 诊前）

用户进来不是"想体验 AI"，而是带着一个具体的小焦虑。这些焦虑全部发生在**一次就医的前后两端**——我们就做这两端的"补充者"，不跟医院抢诊断、不跟阿福抢闭环。

### 场景一 · 诊后：报告解读与用药建议（已去过医院）
- **用户状态**：已做检查、已开药，但**医生回答太简单、没解开疑惑**。
- **典型说法**："医生就说没事，但这几个箭头到底啥意思""这两个药能一起吃吗"。
- **我们的价值**：做医生没空说清楚的那部分——**详细的多专科报告解读 + 个性化用药建议**。
- **为什么强**：用户已有明确数据（报告/药名），输入质量高、AI 发挥空间大；定位"补充"而非"替代医生"，监管与信任风险都低。

### 场景二 · 诊前：预诊与咨询建议（还没去医院）
- **用户状态**：身体不舒服（如头疼），**不想/不确定要不要直接去医院**。
- **典型说法**："我这两天头疼，要不要紧，该看哪个科"。
- **我们的价值**：记录症状 → 多 Agent 协同做**初步分析 + 分诊建议**（建议挂神经科 / 或中医等）+ 红旗预警。
- **为什么强**：多 Agent 共识在"分诊+多视角研判"上最能体现价值；且天然引出"看哪个科/中医"的下一步。

> 两个场景共享同一个底座（多 Agent 共识 + 健康档案），覆盖"去医院前 → 去医院后"的完整链路。

## 3. 信息架构（极简：一个大对话框 + 两个场景入口）

原则：**对话框为主，入口克制。** 不堆 4 宫格，只保留对应两个核心场景的两个入口，其余能力（用药、分诊）作为场景内的下一步，不占首页。

```
┌─────────────────────────────────────────┐
│  顶部右：我的健康档案 · 家人切换（轻量）    │
│                                           │
│             你好，我能帮你看报告           │
│             或先聊聊哪里不舒服             │
│                                           │
│   ┌───────────────────────────────────┐  │
│   │  大对话框                          │  │
│   │  "上传报告，或说说哪里不舒服…"     │  │
│   │                      [📎上传] [↑]  │  │
│   └───────────────────────────────────┘  │
│                                           │
│   [📄 帮我解读报告/用药]  [🩺 我不舒服，先问问] │
│        ↑诊后                  ↑诊前         │
│                                           │
│  底部：最近会诊 / 健康提醒（基于档案，可空） │
└─────────────────────────────────────────┘
```

- **大对话框 = 万能入口**：直接打字或上传报告，自动判断走"诊后解读"还是"诊前预诊"。
- **两个场景入口** = 给不知道怎么开口的用户一个明确起点；点进去即进入有边界的对应流程。
- **用药建议**内嵌在"诊后"流程（解读报告后/拿药后追问），**分诊（科室/中医建议）**是"诊前"流程的产出，**都不单独占首页入口**。
- **健康档案**收在顶部轻量入口，不抢对话框的视觉中心。

## 4. 核心功能模块（按两个场景组织）

> 红线：所有场景**只呈现信息、选项与不确定性，不下诊断、不开处方**；高风险信号触发"建议线下就医"。

### 场景一 · 诊后：报告解读 + 用药建议

**4.1 报告解读**
- **用户路径**：拍照/上传报告 → OCR 结构化 → 多专科 Agent 各自解读 → 共识卡（关键异常 / 需关注 / 可观察）+ 分歧与不确定项 → 追问 或 存入档案。
- **多 Agent 价值**：同一张报告，内分泌看代谢、心内看血脂血压、检验 Agent 看指标可信度、健康导航把术语翻成人话——**多视角联合解读**，比单模型更全，并暴露"哪些指标专家也拿不准"。
- **"补医生"钩子**：针对"医生说没事但没解释"的痛点，主动解释每个异常箭头的含义、是否需要关注、生活建议。
- **差异化**：自动与历史报告对比（升/降/稳趋势），结合档案做个性化解读。

**4.2 用药建议（与报告解读同属诊后，承接）**
- **用户路径**：拍药盒/输入药名（或从解读结论延伸）→ 适应症、用法、禁忌、相互作用 → **结合档案个性化提醒**（如"你有 XX 史，注意…"）。
- **多 Agent 价值**：药学 Agent + 相应专科 Agent + 循证 Agent 交叉校验相互作用与禁忌。
- **监管边界**：用药科普与提醒，不替代处方。

### 场景二 · 诊前：预诊 + 咨询/分诊建议

**4.3 预诊会诊**
- **用户路径**：描述症状 → 多轮追问收敛 → 多 Agent 协同研判 → 共识（可能方向 + 该做什么 + 红旗预警）→ 可 @ 某位专家追问 → 重新共识。
- **复用现有 Demo 的会诊室交互。**
- **监管边界**：初步分析/科普/红旗预警，不下诊断。

**4.4 分诊建议（预诊的产出，非独立入口）**
- 基于预诊结论，建议**该挂哪个科**（如神经内科）；**中医作为可选方向自然出现在此**（如"也可考虑中医调理"，见 §6）。
- （接入资源后）对接挂号 / 在线问诊；MVP 先做"推荐科室"。

### 贯穿两个场景
- **健康档案**为两个场景共同提供上下文与记忆联动（见 §5）。
- 两个场景入口 + 大对话框构成首页（见 §3），其余能力均内嵌于场景，不单独占位。

## 5. 差异化功能：纵向个人健康档案 + 记忆联动（核心壁垒之一）

> 这是把"一次性工具"变成"长期健康伙伴"的关键，也是阿福们做得浅的地方。

### 5.1 档案构成
- **结构化既往史**：疾病史、过敏史、家族史、长期用药、手术史。
- **报告库**：历次体检/检验报告（结构化存储，支持趋势）。
- **健康小日记**：每日不适症状打卡（部位/程度/诱因/持续时间），可接入可穿戴设备数据。
- **家人档案**：多成员管理（借鉴阿福）。

### 5.2 记忆联动（杀手锏体验）
- **跨时间症状关联**：用户再次描述不适时，AI 主动检索历史——
  > "你 5 月 12 日也记录过类似的右上腹隐痛，当时还伴随…；结合这次，可能与 XX 相关，建议…"
- **趋势驱动的主动提醒**：基于报告/日记趋势，主动推送（如"近三次血糖持续上升，建议复查"）。
- **会诊上下文注入**：每次多 Agent 会诊自动带入档案，使共识"个性化"而非"通用科普"——**这是单次问诊产品给不了的**。

### 5.3 共识引擎 × 档案的协同
- 档案为共识提供 **grounding 与个性化锚点**（缓解我们"因果验证弱"的短板：用真实历史数据约束推断）。
- 记忆联动的"病因推断"必须**标注为假设 + 不确定性**，并给红旗预警，严守不诊断红线。

## 6. 中医结合点（差异化 C）

讯飞已有"中医辨证"，所以中医本身不稀奇；**我们的角度是把中医做成共识面板里的一个独立视角**：

- **中西医双视角会诊**：在症状会诊/调理类场景，加入一个"**中医辨证 Agent**"，与西医 Agent 同台，呈现两种思路（如西医查指标/排器质性问题，中医谈体质/调理方向）的**异同与互补**。
- **天然契合点**：中医本身讲究"辨证"和"君臣佐使"的多因素权衡，与多 Agent 辩论求共识的机制同构；中医不同流派的分歧也适合"透明呈现"。
- **最适合的场景**：亚健康/调理/慢病养护（如睡眠、脾胃、疲劳），而非急症/器质性疾病。
- **必须正视的风险**：
  - 中医证据等级与西医不同，循证 Agent 需用不同标准评估，**不能把中医结论包装成"已验证"**；
  - 涉及中药需强合规（药物相互作用、剂量、毒性）；
  - 定位为"调理建议/科普"，绝不替代诊断与处方。
- **建议**：作为 **可选 lens / 后续迭代**，MVP 先验证西医共识，再以"中西医对照"作为差异化亮点上线。

## 7. 多 Agent 共识的产品化呈现（怎么让用户"看得见"）

技术再好，用户无感知就白搭。呈现要点：
- 报告解读/会诊结论页统一用 **共识卡**：✅ 已达成共识 / ⚠️ 分歧与不确定 / 📋 结构化选项。
- 关键结论可展开看"**哪几位专家怎么说、依据是什么**"（透明度=信任）。
- 允许用户 **@ 单个专家追问** 并触发"重新达成共识"（复用现有 Demo 交互）。
- 始终诚实标注不确定性——**这是与单模型"过度自信"产品的核心区隔**。

## 8. MVP 范围与优先级

**MVP（第一阶段，验证两个场景 + 差异化）**
- P0 · 诊后：报告解读（多 Agent 共识 + 历史趋势对比）
- P0 · 诊前：预诊会诊（复用会诊室 Demo）+ 分诊建议（推荐科室，含"可考虑中医"轻提示）
- P0 · 底座：健康档案（既往史 + 报告库 + 症状日记）+ 记忆联动（基础版：跨时间症状关联）
- P1 · 诊后：用药建议（结合档案）
- P1：首页大对话框的自动场景路由（诊后/诊前自动判别）

**非目标（明确不做 / 后做）**
- 不做诊断结论、不开处方、不做影像 AI 读片（避监管）
- 不做医保支付/挂号自建闭环（拼不过阿福，后续走生态对接）
- 完整中医辨证 Agent / 中西医双视角：第二阶段（MVP 仅在分诊处轻提示"可考虑中医"）
- 设备数据深度同步：第二阶段

## 9. 关键指标（北极星与过程指标）
- 北极星：**月度活跃健康档案数**（有持续记录行为的用户，而非一次性问答 UV）。
- 报告解读完成率 / 解读后追问率（衡量共识形态的吸引力）。
- 记忆联动触发率与采纳率（衡量纵向档案价值）。
- 7 日 / 30 日留存（验证"长期伙伴"定位）。
- 共识"分歧标注"对用户信任度的影响（用户调研/NPS）。

---

## 10. 风险与合规
- **监管红线**：全程不诊断、不处方；显著免责声明；高风险红旗强提示并引导线下。
- **数据合规**：健康数据加密、最小化采集、明确授权、可删除（个人信息保护法）；家人档案需授权。
- **因果验证短板**：记忆联动的病因推断一律标"假设+不确定"，并用档案真实数据做约束，避免过度推断。
- **中医合规**：中药相关强校验，定位调理科普。
- **共识可信度**：需用错误重合度等指标证明多 Agent 真带来增益（详见引擎压力测试方案），否则差异化叙事不成立。
